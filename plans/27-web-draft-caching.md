# Web _render_drafts reads file content for every draft on every 2-second poll

> **Depends on:**

## Problem

In `web.py:272`, `_render_drafts` reads the first 120 bytes of every draft file to check for `INNOVATOR_MARKER`:

```python
for path in drafts:
    name = path.stem[len("draft-"):]
    is_innovator = INNOVATOR_MARKER in path.read_text(encoding="utf-8")[:120]
```

This was the exact pattern fixed in `dashboard.py` (commit `fa99861`) and `loop.py` (which now use glob patterns instead of reading content). But `web.py` was not updated. This runs on every 2-second HTMX poll via `_render_main` → `_render_drafts`.

The purpose is to display an "(innovator)" tag next to drafts generated by the innovator. Currently, the innovator is the only source of `draft-*.md` files (manual drafts don't exist in the workflow), making the content check redundant.

## Solution

Since all `draft-*.md` files are created by the innovator (via `_write_plans` in `innovate.py:434-451`), and user-created plans don't use the `draft-` prefix, simply treat all drafts as innovator-generated and remove the file read:

```python
for path in drafts:
    name = path.stem[len("draft-"):]
    tag = ' <span class="dim">(innovator)</span>'
```

If future requirements need to distinguish manual drafts from innovator drafts, this can be revisited by checking for the marker only on first load (not every 2s poll).

## Scope

- `foreman/web.py` — remove `read_text` call in `_render_drafts`, treat all drafts as innovator-generated

## Risk Assessment

Trivially safe. All draft files currently come from the innovator. If a user manually creates a `draft-*.md` file, it gets the "(innovator)" label — a cosmetic inaccuracy, not a functional issue.
